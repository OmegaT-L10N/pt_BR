Começando a usar o OmegaT

Vídeo de CATgutuEN (Dominique Pivard)

O OmegaT é uma ferramenta de memória de tradução livre escrita em Java. É um software destinado a tradutores profissionais. Ele não traduz para você! (O nome disso é "tradução automática" e terá que procurar em outro lugar.)

Funcionalidades profissionais

  • Correspondências parciais
  • Propagação de Correspondências
  • Processamento simultâneo de projetos com diversos arquivos
  • Uso simultâneo de mais de uma memória de tradução
  • Glossários de usuários com reconhecimento de formas flexionadas

Trabalhe com mais de 30 formatos de arquivo

  • Microsoft Word, Excel, Power
  • XHTML and HTML
  • Open Document Format (LibreOffice, OpenOffice.org)
  • MediaWiki (Wikipedia)
  • Texto simples
  • ... e cerca de 30 outros formatos de arquivo

Ferramentas poderosas

  • Suporte Unicode (UTF-8): pode ser usado com alfabetos não-latinos
  • Suporte para idiomas escritos da direita para a esquerda
  • Verificador ortográfico completo
  • Compatível com outras ferramentas de memória de tradução (TMX, TTX, TXML, XLIFF, SDLXLIFF)
  • Possui interface com o Google Translate

O projeto OmegaT e você

OmegaT não é apenas um produto, é um projeto emocionante, e você pode fazer parte disso!

OmegaT é um software livre. Isso significa que usuários como você são livres para baixar e usar o OmegaT, com muito poucas restrições. Você também poderá copiar e distribuir o OmegaT para outras pessoas.

O OmegaT é um programa com código aberto. Isso significa que se você quiser muito, você pode até modificar OmegaT para se ele adeque às suas nececidades. Para os defensores dos programas de código aberto, essa liberdade é ainda mais importante do que não pagar para utilizá-lo.

O projeto OmegaT precisa do apoio de voluntários em quase todas as áreas. Embora muitos usuários saibam disso, a maioria deles não ajuda o porque não sabe programar. O programador desempenha, sem sombra de dúvida, a função mais importante do projeto, pois sem programação não teríamos nenhum programa. Mas programar não é o único trabalho a ser realizado. Abaixo temos uma lista de áreas para as quais outros voluntários seriam mais do que bem-vindos!

Criação de documentação

Criar documentaçãoé parte importante do processo. Muitos de vocês, que estão lendo isso, traduzirão muita documentação para usuários. Você mesmo, já pensou em escrever alguma documentação? É uma experiência realmente reveladora!

Localização

Graças a trabalhos anteriores de tradutores voluntários, a interface e os textos de ajuda do OmegaT foram traduzidos para vários idiomas. Isso significa que só sobraram alguns milhares de outros idiomas! A localização do OmegaT para outros idiomas também é uma tarefa contínua, uma vez que novos recursos e funções estão sendo adicionados o tempo todo. Acesse Instruções sobre como Localizar o OmegaT para ter mais detalhes sobre o a localização/tradução do programa.

Testes

Uma das coisas mais úteis que os usuários OmegaT podem fazer é ... usar o OmegaT. E quando eles acharem erros ou deficiências na documentação, devem nos informar.

Publicidade

Você já pensou em escrever um artigo para a revista da sua associação profissional? Caso contrário, por que não escrever um artigo sobre o uso do OmegaT?

Doações

Se você achou o OmegaT útil e gostaria de nos agradecer fazendo uma doação através da nossa conta no Paypal.

Programação (Java)

OmegaT é escrito em Java. Se você é um programador e conhece Java, verifique a documentação do Código fonte em /docs_devel para ter mais informações sobre como começar.

Programação (outras linguagens)

Se você programa em outra linguagem, você pode nos ajudar escrevendo código que possa ser utilizado pela aplicação principal. Muito código auxiliar já foi escrito em Javascript, Groovy e Tcl / Tk, por exemplo.

"Mas não sei programar nada", você diz. Bem, você já pensou em aprender? Se você já está um tanto quanto familiarizado com os computadores em geral e escolher uma linguagem de script direta como Javascript ou Tcl/Tk, você pode aprender o suficiente para poder escrever scripts simples e úteis em apenas algumas semanas. Desenvolvimento profissional contínuo é o que há!
